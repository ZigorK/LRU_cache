# LRU Cache Implementation

## Краткие сведения о реализации

- **Версия C++**: C++17
- **Сторонние библиотеки**: не использовались (только STL)
- **Дополнительные библиотеки в тестах**: pthread (для многопоточности)

## Реализованные возможности

### Часть 1 
- Получение элемента (`get`)
- Добавление элемента (`insert`)
- Получение размера (`size`)
- Сброс кэша (`clear`)

### Часть 2 
- Произвольные типы ключа и значения (шаблоны)
- Удаление элемента (`remove`)
- Многопоточное чтение (`shared_mutex`)

## Допущения, сделанные в ходе выполнения

1. Выбран алгоритм **приблизительного LRU** (проверка 4 последних элементов)
2. Время доступа измеряется в операциях (атомарный счетчик), а не в реальном времени
3. При вытеснении удаляется не гарантированно самый старый, а один из последних 4
4. Порог "старости" = 1000 операций (эмпирическое значение)
5. Многопоточность реализована через `shared_mutex` (чтение — shared, запись — exclusive)

## Тестирование

В `main.cpp` реализованы:
- Модульные тесты всех операций
- Тест с разными типами данных
- Многопоточный стресс-тест (10 потоков, 5 секунд)

Результаты стресс-теста:
- 0 критических ошибок
- 0 утечек памяти
- Параллельное чтение работает

## Идеи по развитию (Часть 3)

### 1. Поддержка хранения миллионов записей в кэше
- Шардирование или сегментирование (разделение на несколько независимых кэшей)
- Пулинг памяти для уменьшения фрагментации

### 2. Использование альтернативного алгоритма вытеснения для кэша
- LFU (Least Frequently Used) — хорош для данных, которые часто запрашивают, но старые популярные элементы занимают место
- ARC (Adaptive Replacement Cache) — адаптивный алгоритм, лучше LRU при сканировании и лучше  LFU при изменении популярности, сложно реализовать

### 3. Ограничение потребляемой памяти кэшем вместе с данными
- При реализацции стоит учитывать не только количество, но и реальный размер данных
- Вытеснять по достижении лимита в байтах
